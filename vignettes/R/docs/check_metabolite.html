check_metabolite {biocad_registry} R Documentation

Check if metabolite exists in database

Description

Searches for a metabolite in the BioCAD registry database by cross-references (xrefs) and exact mass. If no direct xref matches are found, falls back to checking common names and synonyms. Returns database ID if found.

Usage

check_metabolite(biocad.registry, compound);

Arguments

biocad.registry

A database connection object representing the BioCAD registry database.

compound

A list containing metabolite information with the following elements: \itemize{ \item{xref - List of cross-references (database identifiers)} \item{formula - Chemical formula string} \item{name - Common name of the compound} \item{synonym - Vector of alternative names/synonyms} }

Details

The function performs the following steps: \enumerate{ \item Extracts and validates chemical formula (calculates exact mass) \item Queries database using cross-reference identifiers (xrefs) \item Performs mass tolerance search (Â±1 Da) \item If xref search fails, calls \code{checkmetabolitesynonym} to check by name and synonyms }

Authors

xieguigang <xie.guigang@gcmodeller.org>

Value

An integer representing the unique database ID of the matched compound. Returns \code{NULL} if no matching metabolite is found in the database.

Examples

#\dontrun{

# Assume biocad_registry is an established database connection
compound <- list(
name = "glucose",
formula = "C6H12O6",
xref = list(KEGG = "C00031", CHEBI = "CHEBI:4167"),
synonym = c("dextrose", "blood sugar")
)
check_metabolite(biocad_registry, compound)

#}
